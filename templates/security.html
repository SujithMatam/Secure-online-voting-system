🔐 Security Features Implemented

This application implements comprehensive security measures across three key areas:

1. Authentication & Authorization

Feature Implementation
Password Hashing bcrypt with 12 rounds of salting — passwords are never stored in plaintext
Role-Based Access Control (RBAC) Two roles: admin and voter. Decorators enforce access: @login_required, @admin_required, @verified_required
Account Lockout After 5 failed login attempts, account is locked for 15 minutes
Session Security HttpOnly cookies, SameSite=Lax, 30-minute session timeout, Cache-Control: no-store
Admin Voter Verification Admin must verify voter identity before they can cast votes
Strong Password Policy Minimum 8 chars, requires uppercase, lowercase, digit, and special character

2. Input Validation & Attack Prevention

Attack Vector Prevention Method
Cross-Site Scripting (XSS) Input sanitization with bleach library + Jinja2 auto-escaping + Content-Security-Policy header
Cross-Site Request Forgery (CSRF) Flask-WTF CSRF tokens on every form — all POST requests validated
NoSQL Injection Input type checking (ensures strings, not objects) + sanitization before DB queries
Brute Force Attacks Rate limiting: Login (5/min), Register (3/min), Voting (10/min) + Account lockout
Clickjacking X-Frame-Options: DENY header prevents embedding in iframes
MIME Sniffing X-Content-Type-Options: nosniff header
Username Enumeration Generic "Invalid credentials" message for both wrong username and wrong password

3. Data Protection

Feature Implementation
Password Storage bcrypt hashing with unique salt per password (12 rounds) — irreversible
Vote Integrity Each vote gets a SHA-256 hash with random nonce — tamper detection
Vote Receipts Unique cryptographic receipt token for each vote for voter verification
HTTPS Enforcement Strict-Transport-Security header forces HTTPS in production
Secure Cookies HttpOnly (no JS access), SameSite=Lax (CSRF protection)
Cache Prevention Cache-Control: no-store prevents sensitive data caching / back-button attacks
Audit Logging All actions logged with timestamp, IP address, user agent — stored in DB + file
